# 项目踩坑总结文档设计

## 目标

将多点导航、NeuPAN 跟踪和 Ackermann 转向配置过程中已经确认的问题整理成可持续维护的项目文档，方便后续开发、复现和运行排查。

## 目录结构

创建独立目录：

```text
docs/troubleshooting/
└── README.md
```

`README.md` 作为统一入口。当前先集中记录已经踩到的坑；未来内容明显增多时，再按多点任务、控制接口、车辆几何和运行诊断拆分子文档。

## 正文范围

文档覆盖：

- 多目标点任务为什么需要分段状态机和人工继续机制；
- RViz 目标点顺序与分段路径可视化；
- ROS 2 transient-local 状态话题的 QoS 与启动排查；
- NeuPAN Ackermann 输出中“转向角”和 ROS Twist `angular.z`“横摆角速度”的语义差异；
- 自行车模型中心转角与左右单轮物理转角的区别；
- `±0.52 rad` 单轮限位对应约 `±0.43 rad` 中心转角和 `1.30 m` 最小半径；
- Reeds-Shepp 换向点附近参考窗口退化和低速卡死的原因链；
- 正确的参数调试顺序、重启要求、诊断话题和当前待验证事项。

## 表达方式

- 使用中文 Markdown。
- 先描述现象，再说明根因、修复方式和验证方法。
- 保留关键公式、参数数值和 ROS 2 命令。
- 明确区分已经修复的问题与仍需运行验证的项目。
- 不复制大段源代码，不修改 README 主入口或任何运行配置。

## 验收标准

- `docs/troubleshooting/README.md` 可以独立阅读。
- 内容与当前配置口径一致：单轮 `±0.52 rad`、中心 `±0.43 rad`、最小半径 `1.30 m`。
- 文档包含多点任务、QoS、控制语义、转向几何、卡死链路、调参顺序和运行检查清单。
- 不执行任何 Git 操作。
